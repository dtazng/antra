# UI Contracts: Day View

**Feature**: `006-day-view`
**Date**: 2026-03-11

These contracts define the observable behavior of each Day View component ‚Äî what inputs they accept, what they display, and how they respond to user actions. They are the basis for widget tests.

---

## Component 1: `RelationshipBriefing`

### Input Contract

| Input | Type | Description |
|---|---|---|
| `suggestions` | `List<Suggestion>` | 2‚Äì4 items generated by `SuggestionEngine`. May be empty. |
| `loading` | `bool` | If `true`, shows a loading skeleton. |

### Display Contract

**When `loading = true`**:
- Show a placeholder skeleton (no text, no suggestion rows).

**When `suggestions.isEmpty` and `loading = false`**:
- Show an encouraging neutral message (e.g., "Your relationships are looking good today.").
- Do NOT show a blank area.

**When `suggestions.isNotEmpty`**:
- Show a greeting header (e.g., "Good morning.").
- Show a subtitle: "Here are [N] relationship things worth doing today:"
- Render one row per suggestion with a bullet point. Each row is a plain human-readable sentence.
  - Birthday: "{name} has a birthday {tomorrow | in N days}."
  - Follow-up: "Follow up with {name} after your meeting."
  - Reconnect: "You haven't talked to {name} in {N} days."
  - Memory: "You met {name} {N} year(s) ago today."
- Maximum 4 rows shown regardless of `suggestions` length.

### Acceptance Test Criteria

1. Given `suggestions = []` ‚Üí single neutral message visible, no blank area.
2. Given `suggestions` with 4 items ‚Üí exactly 4 rows displayed.
3. Given `suggestions` with 2 birthday items ‚Üí each row contains the contact name.
4. Given `loading = true` ‚Üí no suggestion text rendered.

## Component 2: `DailyGoalWidget`

### Input Contract

| Input | Type | Description |
|---|---|---|
| `goal` | `DailyGoal` | Contains `target` (int), `reached` (int), `completed` (bool). |

### Display Contract

**When `goal.completed = false`**:
- Header text: "Reach out to {target} people today"
- Progress text: "{reached} / {target} completed"
- A linear progress indicator from 0.0 to 1.0 (`reached / target`).

**When `goal.completed = true`**:
- Replace entire section with completion message: "Daily relationships complete ‚úì"
- Subtitle: "You strengthened {reached} connections today."
- Progress bar is hidden or shown as full.

### Acceptance Test Criteria

1. Given `DailyGoal(target: 3, reached: 1, completed: false)` ‚Üí shows "1 / 3 completed".
2. Given `DailyGoal(target: 3, reached: 3, completed: true)` ‚Üí shows completion message.
3. Given `DailyGoal(target: 3, reached: 0, completed: false)` ‚Üí shows "0 / 3 completed", empty progress.

## Component 3: `SuggestionCard`

### Input Contract

| Input | Type | Description |
|---|---|---|
| `suggestion` | `Suggestion` | The suggestion to display. |
| `expanded` | `bool` | Whether the card is currently in expanded state. |
| `onTap` | `VoidCallback` | Called when the card header area is tapped. |
| `onAction` | `void Function(SuggestionAction)` | Called when an action button is tapped. |
| `onDismiss` | `VoidCallback` | Called when the card is dismissed (swiped or close tapped). |

### Collapsed Display Contract

- Contact name (prominent).
- Card type chip: "Reconnect" | "Birthday üéâ" | "Follow-up" | "Memory" | "Recent".
- Signal text (e.g., "Last contact: 32 days ago").
- No action buttons visible.

### Expanded Display Contract (animated in, 250ms ease-in-out)

- All collapsed content remains visible.
- Notes section: shows `suggestion.personNotes` if non-empty, otherwise omitted.
- Action row with 2‚Äì4 buttons depending on card type:

| Card Type | Actions Available |
|---|---|
| Reconnect | Message ¬∑ Call ¬∑ Log meeting |
| Birthday | Send greeting ¬∑ Log call |
| Follow-up | Follow up ¬∑ Schedule later ¬∑ Mark done |
| Memory | Log note ¬∑ Message |

### Post-Action Behavior

- On any action tap: show brief success state (e.g., text changes to "‚úì Logged"), then the card collapses and is removed from the feed after 800ms.

### Acceptance Test Criteria

1. Given `expanded = false` ‚Üí action buttons NOT in widget tree.
2. Given `expanded = true` ‚Üí action buttons ARE in widget tree.
3. Given `expanded = false`, tap card ‚Üí `onTap` called.
4. Given `expanded = true`, tap "Log meeting" ‚Üí `onAction(SuggestionAction.logMeeting)` called.
5. Given card type `birthday` ‚Üí action buttons include "Send greeting".
6. Given card type `reconnect` ‚Üí action buttons include "Message", "Call", "Log meeting".
7. Given `suggestion.personNotes = null` ‚Üí notes section absent from expanded view.

## Component 4: `QuickLogBar`

### Input Contract

| Input | Type | Description |
|---|---|---|
| `onInteractionLogged` | `void Function(String bulletId)` | Called after a successful save. |

### Display Contract

- Always visible, pinned to the bottom of the Day View screen.
- Shows 4 tappable icons with labels: ‚òï Coffee | üìû Call | ‚úâÔ∏è Message | ‚úçÔ∏è Note.

### Interaction Flow Contract

**Step 1 ‚Äî Tap interaction type**:
- The tapped type is visually selected (highlighted).
- A person-picker UI appears (bottom sheet or inline search). Shows existing contacts sorted by recency.

**Step 2 ‚Äî Select person**:
- Person picker shows a searchable list of existing contacts.
- Tapping a contact selects them and advances to Step 3.

**Step 3 ‚Äî Save (optional note)**:
- The selected type + person name are visible.
- An optional note field is available but empty.
- A "Save" button is tappable.
- Tapping "Save" without a note: logs immediately.

**On save**:
- Bullet is created: `type = 'event'` (or 'note' for Note type), content = auto-generated or user-typed.
- `bullet_person_links` entry is created.
- `onInteractionLogged(bulletId)` called.
- UI resets to idle state.

**3-tap minimum path**: type tap ‚Üí person tap ‚Üí Save tap = exactly 3 taps, no note required.

### Acceptance Test Criteria

1. Given idle state ‚Üí 4 type icons visible, no person picker.
2. Given type tapped ‚Üí person picker appears.
3. Given person selected ‚Üí Save button becomes available.
4. Given Save tapped (no note) ‚Üí `onInteractionLogged` called; UI resets.
5. Given type tapped, then Cancel ‚Üí UI returns to idle, no data written.
6. Given Note type selected ‚Üí `type = 'note'` bullet created; content field required (must not be empty).

## Component 5: `TodayInteractionTimeline`

### Input Contract

| Input | Type | Description |
|---|---|---|
| `interactions` | `List<TodayInteraction>` | Today's logged interactions, newest first. |
| `onTap` | `void Function(String bulletId)` | Called when an entry is tapped. |

### Display Contract

**When `interactions.isEmpty`**:
- Empty-state message: "No interactions logged yet today."
- Do NOT show a blank area.

**When `interactions.isNotEmpty`**:
- Section header: "Today"
- Each entry shows: timestamp (HH:mm), interaction label + contact name.
  - Example: "09:20 ‚Äî Coffee with Alex"
- Entries are sorted newest-first (top = most recent).

### Acceptance Test Criteria

1. Given `interactions = []` ‚Üí empty-state message visible.
2. Given 3 interactions ‚Üí 3 entries visible in reverse-chronological order.
3. Given entry tapped ‚Üí `onTap(bulletId)` called with correct ID.
4. Given Coffee interaction ‚Üí entry displays "‚òï Coffee with [name]" or equivalent label.

## Component 6: `DayViewScreen` (orchestrator)

### Behavior Contract

- Rendered as Tab 0 of `RootTabScreen`.
- Scrollable: `RelationshipBriefing` ‚Üí `DailyGoalWidget` ‚Üí `SuggestionCard` feed ‚Üí `TodayInteractionTimeline`. `QuickLogBar` is pinned to bottom, above the floating tab bar.
- When a suggestion card action logs an interaction:
  1. The interaction appears in the `TodayInteractionTimeline`.
  2. The `DailyGoalWidget` progress increments.
  3. The suggestion card is removed from the feed.
- When `QuickLogBar` logs an interaction:
  1. Same as above (points 1‚Äì2).
  2. If the logged person has a suggestion card, that card is removed from the feed.
- At most one `SuggestionCard` is expanded at any time.
- The `QuickLogBar` does not navigate away from the Day View screen.

### Acceptance Test Criteria

1. Given Day View loads ‚Üí all 5 sections rendered (briefing, goal, cards, log bar, timeline).
2. Given one card expanded, another card tapped ‚Üí first collapses, second expands.
3. Given Quick Log saves an interaction ‚Üí timeline updates without screen navigation.
4. Given Quick Log person matches a suggestion card ‚Üí that card removed from feed.
5. Given goal reached (3 interactions logged) ‚Üí completion state shown in `DailyGoalWidget`.
