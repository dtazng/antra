# Implementation Plan: Log UX Refinement

**Branch**: `008-log-ux-refine` | **Date**: 2026-03-13 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/008-log-ux-refine/spec.md`

## Summary

Refine the logging UX of the Day View: fix the input card's corner radius visual bug, add clear task vs note visual distinction in the timeline feed, improve the type toggle with labels and subtitles, expand the people-linking UI to support multiple people per log entry, and add swipe-to-delete with undo. No new packages or DB migrations required — the `bullet_person_links` table already supports multi-person at the DB level; all changes are confined to widgets, one DAO addition, and one widget contract extension.

## Implementation Strategy

### MVP: US1 + US2 (corner fix + task/note distinction)

Both require no API changes and no callers to update. They are fully independent. Ship these first to unblock visual regression and establish the type identity pattern.

### Phase 2: US3 (type switch labels)

One widget change (`BulletCaptureBar` type toggle area). No downstream effects. Can ship independently.

### Phase 3: US4 (multi-person linking)

Requires coordinated changes across `PersonPickerSheet` (return type), `BulletCaptureBar` (chips, submission logic), and integration tests. Depends on no other story — can be developed in parallel with US3 but tested after both are merged.

### Phase 4: US5 (swipe-to-delete)

Requires `undoSoftDeleteBullet` DAO method and `onDelete` callback on `TodayInteractionTimeline`. `DayViewScreen` wires the callback and shows the snackbar. Depends on no other story in this feature but requires the updated `TodayInteractionTimeline` contract.

## Key Implementation Details

### 1. GlassSurface borderRadius override

In `_GlassSurfaceState.build()`, replace `props.borderRadius` with `widget.borderRadius ?? props.borderRadius` in both the `Container` decoration and the `ClipRRect`. Pass `borderRadius: BorderRadius.circular(AntraRadius.card)` from `BulletCaptureBar`.

### 2. Type toggle — labeled area

Replace the current `GestureDetector + Icon` type toggle in `BulletCaptureBar` with a `Column` containing:

- `Text(_selectedType == 'note' ? 'Note' : 'Task', fontSize: 13, color: white70)`
- `Text(_selectedType == 'note' ? 'Context' : 'Follow-up', fontSize: 10, color: white38)`

Tap zone: same `GestureDetector` wrapping both texts.

### 3. Task vs note timeline indicator

In `TodayInteractionTimeline._buildEntry`, update the leading icon logic:

- `type == 'task'` → `Icons.check_box_outline_blank_rounded`, size 12, white54
- `type != 'task'` (note/default) → `Icons.circle`, size 6, white38

Append `TASK` label as `Text('TASK', fontSize: 10, color: white38, letterSpacing: 0.8)` when `type == 'task'`, inside the row after the content `Expanded`.

### 4. Multi-person chips in BulletCaptureBar

Change `PeopleData? _linkedPerson` → `List<PeopleData> _linkedPeople = []`.

Chip row (above text field, shown when `_linkedPeople.isNotEmpty`):

```dart
Wrap(
  spacing: 6,
  children: _linkedPeople.map((p) => _PersonChip(person: p, onRemove: ...)).toList(),
)
```

`_PersonChip`: small `PersonAvatar(radius: 10)` + name text + `×` icon, styled like the current single-person chip but built from a `Wrap` for multiple.

### 5. PersonPickerSheet multi-select

Add `List<PeopleData> _selected = []` state, initialized from `widget.alreadyLinked`.

Each person row gets a trailing `Icon(Icons.check_rounded, color: white70)` when selected. Tapping toggles.

Add a "Done" `TextButton` in the sheet header row (right-aligned): `Navigator.of(context).pop(_selected)`.

Dismissing without tapping Done pops `widget.alreadyLinked` unchanged (handled by `WillPopScope` or `onPopInvoked`).

### 6. Swipe-to-delete in TodayInteractionTimeline

Wrap each `_buildEntry` result in a `Dismissible`:

```dart
Dismissible(
  key: ValueKey(entry.bulletId),
  direction: DismissDirection.endToStart,
  background: Container(
    alignment: Alignment.centerRight,
    padding: const EdgeInsets.only(right: 20),
    color: Colors.red.shade800,
    child: const Icon(Icons.delete_outline_rounded, color: Colors.white),
  ),
  onDismissed: (_) => widget.onDelete(entry.bulletId),
  child: _buildEntryContent(...),
)
```

`DayViewScreen` receives the `onDelete` callback, calls `BulletsDao.softDeleteBullet(id)`, then shows:

```dart
ScaffoldMessenger.of(context).showSnackBar(
  SnackBar(
    content: const Text('Entry deleted'),
    action: SnackBarAction(label: 'Undo', onPressed: () => bulletsDao.undoSoftDeleteBullet(id)),
    duration: const Duration(seconds: 4),
  ),
);
```
